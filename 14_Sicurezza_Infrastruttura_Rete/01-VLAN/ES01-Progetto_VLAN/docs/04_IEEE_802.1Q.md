# IEEE 802.1Q - VLAN Tagging

## Introduzione

**IEEE 802.1Q** è lo standard internazionale che definisce il protocollo per il **VLAN tagging** su reti Ethernet. È il meccanismo più utilizzato per trasportare traffico di multiple VLAN su un singolo link fisico.

## Storia

- **Anno**: 1998 (prima versione)
- **Compatibilità**: Tutti i vendor di networking
- **Utilizzo**: Standard de facto per VLAN tagging

## Struttura del Frame 802.1Q

### Frame Ethernet Standard (senza tag)
```
┌─────────────┬──────────────┬──────────┬──────────┬─────┐
│ Dest MAC    │ Source MAC   │   Type   │   Data   │ FCS │
│  (6 byte)   │   (6 byte)   │ (2 byte) │(46-1500) │(4B) │
└─────────────┴──────────────┴──────────┴──────────┴─────┘
Total: 64-1518 byte
```

### Frame 802.1Q Tagged
```
┌─────────────┬──────────────┬────────────────┬──────────┬──────────┬─────┐
│ Dest MAC    │ Source MAC   │  802.1Q TAG    │   Type   │   Data   │ FCS │
│  (6 byte)   │   (6 byte)   │   (4 byte)     │ (2 byte) │(46-1500) │(4B) │
└─────────────┴──────────────┴────────────────┴──────────┴──────────┴─────┘
Total: 68-1522 byte (+4 byte per il tag)
```

### Dettaglio 802.1Q Tag (4 byte)

```
┌──────────────────┬──────┬─────┬────────────────────┐
│       TPID       │ PCP  │ DEI │        VID         │
│     16 bit       │ 3bit │ 1bit│      12 bit        │
└──────────────────┴──────┴─────┴────────────────────┘
```

**Campi del Tag:**

#### TPID (Tag Protocol Identifier) - 16 bit
- **Valore**: 0x8100
- **Scopo**: Identifica il frame come 802.1Q tagged

#### PCP (Priority Code Point) - 3 bit
- **Valori**: 0-7
- **Scopo**: Quality of Service (QoS)

| Valore | Priorità | Tipo Traffico |
|--------|----------|---------------|
| 0 | Best Effort | Traffico normale |
| 1 | Background | Backup |
| 5 | Voice | < 10ms latency |
| 7 | Network Control | Massima priorità |

#### DEI (Drop Eligible Indicator) - 1 bit
- **Valore 0**: Frame non eligible per il drop
- **Valore 1**: Frame può essere scartato in caso di congestione

#### VID (VLAN Identifier) - 12 bit
- **Range**: 0-4095 (2^12 = 4096 valori)
- **Identifica la VLAN** a cui appartiene il frame

**Valori riservati:**
- **0**: Priority-tagged (no VLAN, solo QoS)
- **1**: Default VLAN
- **4095**: Riservato

**Valori utilizzabili:**
- **1-1005**: Normal range VLAN
- **1006-4094**: Extended range VLAN

## Funzionamento del Tagging

### Porta Access (Untagged)

**Traffico in ingresso:**
1. Frame arriva senza tag 802.1Q
2. Switch aggiunge tag con VID della VLAN associata
3. Frame tagged viene inoltrato

**Traffico in uscita:**
1. Switch riceve frame tagged
2. Rimuove il tag 802.1Q
3. Invia frame standard al dispositivo

### Porta Trunk (Tagged)

**Traffico in ingresso:**
1. Frame può arrivare con o senza tag
2. Se tagged: mantiene il tag
3. Se untagged: assegnato alla Native VLAN

**Traffico in uscita:**
1. Frame della Native VLAN: senza tag
2. Frame di altre VLAN: con tag 802.1Q

## Native VLAN

La **Native VLAN** è la VLAN il cui traffico viaggia **senza tag** su una porta trunk.

**Caratteristiche:**
- Default: VLAN 1
- Deve essere identica su entrambi i lati del trunk
- Best practice: usare VLAN dedicata e inutilizzata

**Security Risk - VLAN Hopping:**
```cisco
! Mitigazione
switchport trunk native vlan 999
vlan dot1q tag native
```

## QoS e 802.1Q

### Priority Tagging (802.1p)
- Campo PCP nel tag: 3 bit = 8 livelli di priorità
- Usato per QoS Layer 2

### Mapping CoS-DSCP

| CoS | DSCP | Traffic Type |
|-----|------|--------------|
| 0 | 0 (BE) | Best Effort |
| 5 | 46 (EF) | Voice |
| 6 | 48 (CS6) | Control |

## Security Considerations

### VLAN Hopping Attack

**Double Tagging Attack:**
1. Attaccante su Native VLAN
2. Crea frame con doppio tag
3. Primo tag rimosso (Native VLAN)
4. Secondo tag permette accesso a VLAN target

**Mitigazione:**
```cisco
! Non usare VLAN 1 come native
switchport trunk native vlan 999

! Disabilitare porte inutilizzate
interface range Fa0/10-24
shutdown

! Disabilitare DTP
switchport nonegotiate
```

## Best Practices

1. **Native VLAN dedicata**: Non usare VLAN 1
2. **Limitare VLAN sui trunk**: Solo VLAN necessarie
3. **Disabilitare DTP**: `switchport nonegotiate`
4. **Documentare**: Schema VLAN aggiornato
5. **QoS planning**: Usare PCP per prioritizzazione
6. **Security**: VLAN hopping prevention
7. **MTU awareness**: Verificare supporto baby giant frames (1522 byte)


## Troubleshooting

### Problema: Trunk non funziona

```cisco
show interfaces GigabitEthernet0/1 switchport
! Verificare:
! - Operational Mode: trunk
! - Encapsulation: dot1q
```

### Problema: Native VLAN mismatch

```cisco
show interfaces trunk
! Verificare Native VLAN identica su entrambi i lati
```

## Conclusioni

802.1Q è lo standard fondamentale per:
- VLAN trunking multi-vendor
- QoS tramite priority tagging
- Scalabilità delle reti moderne
- Flessibilità nella progettazione

La comprensione di 802.1Q è essenziale per configurazione, troubleshooting e sicurezza delle reti enterprise.
